# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- `npm run dev` — start Nuxt dev server on `http://localhost:3000`. **Do not run this yourself** — the user starts the dev server to test functionality.
- `npm run build` — production build.
- `npm run generate` — static site generation.
- `npm run preview` — preview production build locally.
- `npm run lint` — run ESLint and Stylelint together (both must pass).
- `npm run lint:fix` — auto-fix ESLint + Stylelint.
- `npm run lint:style` / `npm run lint:style:fix` — Stylelint only.
- `postinstall` runs `nuxt prepare` automatically; a Husky `pre-commit` hook runs `lint-staged` (ESLint on `*.{js,ts,vue}`, Stylelint on `*.{css,vue}`).

There is no test runner wired up despite `@nuxt/test-utils` being installed.

## Architecture

Single-page Nuxt 3 app (Vue 3, Composition API) that renders a 3D CSS cube with animated "sparkle" decorations on each face. The project is intentionally small; most interesting logic is CSS-driven.

- `app.vue` → `<NuxtPage />` → `pages/[[side]].vue` mounts a single `TheCube` with a `ProfileCard` in the `front` slot. The URL is the source of truth for the visible side: `/1`../`/6` map to `SIDES` positions (`utils/sides.ts`), `/` means `front`, anything else 404s via `validate`. Each spin does `router.push`, so browser back/forward replays rotations; the optional-param route keeps one component instance so rotations animate instead of remounting. The page background behind the cube is per-side (`SIDE_BACKGROUNDS` in `utils/sides.ts`: front `#c05667`, back aqua `#95efff`, bottom its own `--color-page-bottom` split off the shared ink for solo tuning, the remaining sides the dark ink `#1b3964`) and crossfades over the same 2.1s as the spin via the `--page-background` custom property on `.main-page`.
- `components/TheCube.vue` is the central widget: a display-only component controlled via the `side` prop that renders six named slots (`front`/`back`/`left`/`right`/`top`/`bottom`) — the `left`/`right`/`top` faces each wrapped with a `SparklesSparkles` of a distinct `type`, while `front`/`back`/`bottom` carry dedicated ambience components instead (see MeteorShower/MeteorGlow, RippleField, GlitchField below). 3D transforms on `.cube[data-side="…"]` drive the rotation; hidden sides get `inert`, and sparkles are mounted only on the active side — constantly repainting edge-on (90°) layers otherwise rasterize as garbage-stripe GPU glitches in Chromium. (A `.bounds-plane` backdrop that pinned the scene's projected bounds for the same reason was removed once the cube got simple enough not to glitch — its 2% ink tint showed as a faint rectangle edge on the bright back-side page.) Rotation is triggered only by navigation — `SideNavigation` dots, the profile `Contacts` button, and arrow keys handled in `pages/[[side]].vue` (they navigate to the geometric neighbor from `SIDE_NEIGHBORS` in `utils/sides.ts`). There is no click-to-spin on the cube itself.
- `components/SparklesSparkles.vue` generates per-type sparkle particles. Positions come from a fixed `SPARKLE_POSITIONS_BASE` table + random offset, re-randomised on each CSS `animationiteration` via `updateSparklePosition`. Type-specific counts are clamped by `MAX_PARTICLES`. When modifying sparkle shapes, note that `star` renders an inline SVG while the others (`cross`/`plus`/`asterisk`/`diamond`) are built entirely from `::before`/`::after` pseudo-elements styled via `--sparkle-*` custom properties.
- `components/MeteorShower.vue` replaces sparkles entirely on the `front` (Profile) side: a page-space meteor rain mounted by `pages/[[side]].vue` as two viewport-covering layers — `layer="back"` swaps in for the mirrored background sparkles behind the cube, `layer="front"` sits after `TheCube` in the DOM and paints over it — so streaks pass both behind and in front of the cube (the `front` face itself renders no `SparklesSparkles`; the `right` face carries the `star` sparkles instead, and streak/head colors all derive from the single palette base `--color-meteor` (pale cyan `#44fbff`) via relative HSL lifts in the component CSS, so the head and tail tip stay saturated rather than fading to white and retuning the base recolors the whole meteor). Streaks fly 25°±3° off vertical (down-left); tilt and translate derive from the same per-meteor angle so each flies along its own tail. All tuning lives in component constants: `LAYER_CONFIGS` (depth-correlated length/width/opacity/blur/travel/duration per layer), weighted spawn zones (mostly off-screen entries from the top/right edges — the zones must not overlap when projected perpendicular to the flight direction, or the double-fed corridor visibly overfills), and a `through`/`burnout` variant split (most meteors get travel stretched to exit the frame still lit; the rest dim out mid-sky). Respawns re-randomise lane/tilt on `animationiteration` but never touch `animation-duration`/`delay` — changing those mid-animation shifts the running phase. The long-term plan is a unique ambient animation per side.
- `components/MeteorGlow.vue` is the light those nearest meteors cast on the cube: mounted inside TheCube's front face after the slot (so the `mix-blend-mode: screen` patches brighten both the poster gradient and the ink typography), one soft elongated patch per front-layer streak. The front MeteorShower instance publishes its streak elements through `composables/useMeteorLightSources.ts` (a module-level `shallowRef`; retraction is owner-checked so a leave-transitioning old shower can't wipe a fresh instance's registration), and MeteorGlow drives the patches in a single rAF loop — all reads first (streak `getBoundingClientRect`, inline `--meteor-tilt`/`--meteor-length`, computed animated opacity), then writes, which are **custom properties, not finished values**: the loop only reports the measured `--glow-head-x`/`--glow-head-y`/`--glow-tilt`/`--glow-alpha` per patch, and the stylesheet composes them into the one `transform`/`opacity` pair (still compositor-only). Every dial of how the light *looks* therefore lives in CSS on `.meteor-glow` — `--glow-length-ratio`/`--glow-width-ratio` size the patch off the streak length JS hands over as `--streak-length`, `--glow-backset-ratio` backs it off up the tail (a `translateY` applied after `rotate`, i.e. in the streak's own axes), `--glow-strength` caps the intensity — so retuning the glow never means touching JS. The head sits at rect center + (length/2)·(−sin θ, cos θ) (CSS clockwise rotation about the streak's center), and page coordinates map into face-local ones through the glow layer's own rect — an affine mapping valid only while the face is screen-parallel, so every frame first compares the projected layer rect against its layout size (±0.5px) and hides the patches mid-spin instead of smearing them over a rotating face — no fixed post-spin timer, and the settle moment doubles as the anchor of the fade-in ramp, which is multiplied into the written `--glow-alpha` rather than transitioned on the layer (an ancestor `opacity < 1` would isolate the screen blending into a transparent group). Streak lengths are static per meteor and read once per publication; off-face patches are culled by zeroing `--glow-alpha`, using one streak length of slack so the cull never has to know the CSS ratios. The shared base color lives in the palette as `--color-meteor`.
- `components/RippleField.vue` is the `back` ("How I work") side's ambience — the "wet print" concept: the side is a CMYK press sheet by acid water, with Cyberpunk 2077 / Marathon as the color reference. Unlike MeteorShower/GlitchField it splits `page`/`face`, not back/front: `layer="page"` is mounted by `pages/[[side]].vue` behind the cube (the pale aqua backdrop water), `layer="face"` is mounted *inside TheCube's back face* (`v-if="side === 'back'"`) under the `WorkflowCard` slot content and is deliberately **still** — print-sheet furniture only (trim frame, crop marks, registration target, CMYK color bar, faint paper grain), no water, no filter, so the ink text reads undisturbed. The page scene is oversized (the bleed must exceed the warp's peak `scale/2` shift) and warped by a live SMIL-morphing `feTurbulence`+`feDisplacementMap` filter (`#ripple-warp-page`, scale 55–90 ≈ ±27–45px) — the honest heavy part, re-rasterizing every frame. The morph avoids visible turnaround moments: `baseFrequency` orbits a closed convex loop in (fx, fy) space (with `keyTimes` matched to segment lengths for constant orbit speed) instead of ping-ponging between two values, so its direction only ever turns; `scale` is one-dimensional and has to come back, so its swell is spline-eased to zero velocity at both extremes — a truly endless one-way morph isn't available, since `feTurbulence` has no phase/time input and `seed` doesn't interpolate (a seed change is a hard cut). The water itself is fully transparent — no colored wave overlays; the warp shows only through what it bends: a large magenta checkerboard (`.checker-sector`, `repeating-conic-gradient` of the side accent `--color-accent-back`) filling the top-right and bottom-left quadrants, each grid anchored to the corner touching the page center so only whole cells grow outward from the center (cells get cut by the screen edge alone) — regular straight strokes exist mostly to make the distortion legible, since displacement is invisible on flat color but obvious on a bent line. (Earlier iterations painted colored caustic webs, a giant outline-text echo of the headline, a registration crosshair and its circle's arcs over the water; all were removed in favor of the transparent-refraction checkers.) `WorkflowCard.vue` carries the matching typography: a Bungee `HOW I WORK` headline with misregistered cyan/magenta plates that slowly orbit the key plate (each plate is a `content: attr(data-text)` copy of the line animated with `rotate → translateX → counter-rotate`, not a `text-shadow` pair — shadow offsets can only step between keyframes and the corners of that polygon read as a stutter; the yellow plate is "invisible on yellow paper"), Space Grotesk principles each tagged with a plate swatch (cyan/magenta/key — yellow would vanish), and tracked-out caps for the tools line.
- `components/GlitchField.vue` is the `bottom` ("???") side's ambient animation, mounted by `pages/[[side]].vue` as the same two page-space layers as MeteorShower (the `bottom` face renders no `SparklesSparkles`). The back layer is dim purple lit strips; the front layer paints static scanlines, a pre-baked feTurbulence noise tile (motion faked via opacity/background-position jumps — the filter never re-runs), and glitch bars that corrupt whatever is painted beneath them via `backdrop-filter`. The front layer also rolls a single interference band over the whole scene at a constant CRT-like pace (painted before the bars so bar flashes corrupt it too). The band drags its own deformation with it: a second page-level filter (`#interference-warp-filter`, applied as `.band-warp` on `.main-page`) chains two displacement passes that both ride the band. Pass one is the latex dent — a pre-baked vertical pull gradient (`feImage`, only G varies; R/B stay 0.5) with its own `feDisplacementMap` (scale 56) that drags the frame down along the band's travel, the peak at 0.62 of the strip so the wake above stretches long while the bow ahead compresses short (the drag read); its dome height reads `--band-dent-height` (default 270) each frame alongside the morph pace. Pass two is the horizontal CRT tear inside a stripe pinned to the band — two live `feTurbulence` fields (x frequency 0, so every row shifts as a whole scanline) crossfaded by an arithmetic `feComposite` whose weights slide along a cosine, so the bend writhes smoothly from one profile into the next while each field re-rolls its seed only at the moment its weight touches zero (never a visible cut); the fields render at a fixed origin and an `feOffset` translates the blended noise to the measured band position each frame, so the bend profile travels with the band instead of being a stationary page-space field the stripe merely reveals (the offset's own primitive subregion is re-pinned to the destination alongside `dy` — left unset it defaults to the union of the inputs' subregions, i.e. the fixed strip at the top, and hard-clips the shifted noise right back there). An `feColorMatrix` contrast-shapes the blend and pins G to 0.5 (the tear never shifts rows vertically), a static `feImage` alpha-gradient mask fades the stripe's ends, and everything lands over a neutral `#808080` `feFlood` — `color-interpolation-filters="sRGB"` on the filter is load-bearing, in linearRGB the flood stops being neutral and the whole frame shifts. A rAF loop measures the band's `getBoundingClientRect` each frame (the CSS roll animation stays the source of truth) and writes the mask `y`, the tear `feOffset` `dy`, the dent `feImage` `y`/`height`, the crossfade weights and the seeds via direct `setAttribute` (filter primitives can't consume CSS vars); the morph pace is measured off `--band-warp-morph-seconds`, and the band's glow/fringe/noise intensities read `--band-glow-opacity`/`--band-fringe-opacity`/`--band-noise-opacity` (fallback defaults live inline in the `var()` usage — no local declarations, so inline overrides on `<html>` inherit through). Activation follows the burst contract: a new `band-warp` emit fires after the same 2.1s arrival grace (the page filter flattens the cube's 3D) and the page gates the class on `side === 'bottom'`; when a burst lands mid-roll both filters chain. Displacement at the frame edges pulls in transparency and exposes what lies beneath `.main-page` — the body base, near-black `#0d0d0d` site-wide (`--color-page-base` in `global.css`), so the edge gaps read as the frame tearing over darkness. The band itself carries chromatic fringes (`::before`/`::after` in `--color-split-cold`/`--color-split-warm` — the pair shared with `LlmNoteCard`'s RGB clones, promoted to `:root`) and a `.band-noise` VHS head-switching strip reusing the shared `--noise-tile` custom property with a stepped `background-position` crawl. Bars stay `visibility: hidden` between flashes so their backdrop-filter surfaces only exist for the few visible frames; flash windows sit at the cycle tail and `animationiteration` respawns line/jitter only (never duration/delay). The front instance also schedules whole-frame displacement bursts (60–110ms every 2–4s): it re-seeds an SVG `feTurbulence`+`feDisplacementMap` filter def it owns and emits `frame-glitch`, and the page applies `filter: url("#glitch-frame-filter")` to `.main-page`, gated on `side === 'bottom'` so a burst never fires mid-spin. `LlmNoteCard.vue` carries matching RGB-split flashes on the title, terminal filename, and several terminal text fragments via a shared `.rgb-flash` utility (`content: attr(data-text)` pseudo clones; per-element cycle/phase/amplitude through `--rgb-flash-cycle`/`--rgb-flash-phase`/`--rgb-flash-shift`, plus a `.rgb-flash-stutter` modifier whose keyframes hit twice per cycle; splits are always horizontal, and targets must stay single-line for the clones to overlay — inline spans were chosen to fit the current line wrapping). It also hides a hacker-typer easter egg: printable keydowns while the side is active (guarded by checking the face's `inert`) reveal a canned tease at the blinking cursor, one character per keypress, Backspace erases.
- `components/ui/` (`Button`, `Text`, `Title`) are shared primitives consumed via Nuxt auto-import as `UiButton`, `UiText`, `UiTitle` (the `Ui` prefix comes from the `ui/` subfolder, per Nuxt's component-name convention). Each lifts itself one elevation level (`translateZ(var(--elevation-step))`) above its parent surface inside the cube's 3D context; `UiText` lifts arbitrary text content and renders its slot twice — the extra copy is projected one level down and repainted via `-webkit-text-fill-color` as a cast shadow; `UiTitle` is the pastel accent-tinted face heading built on `UiText`. The design is deliberately minimal: faces are flat solid `--surface-color` panels, there are no borders, gradients or filled cards (only the purple side's terminal keeps its own panel), and actions/links are bare text that answers hover with a pastel `--side-accent` tint plus an elevation lift. The `front` (Profile) face is the poster-style exception: its surface is a radial gradient from the accent `#f7abbf` at the center to `--color-front-corner` `#bf5368` at the corners, typography sits on it in `--side-ink` (`#1b3964`) heavy display caps (`--font-poster` Archivo Black) with condensed support lines (`--font-condensed` Oswald), and its actions hover to white instead of the pastel tint; its `--surface-color` is a `color-mix()` of the gradient endpoints (70% accent / 30% corner) so `UiText` cast shadows keep a plausible local tone. The `back` (How I work) face is the second bright exception — the wet-print sheet: a deliberately uniform pale acid-yellow solid (`color-mix` of `--color-back-paper` toward white, kept flat and still so the text stays readable) with `--side-ink` typography in the print pair (`--font-signage` Bungee, `--font-grotesk` Space Grotesk; all four families load via one Google Fonts link in `nuxt.config.ts`).
- `components/ColorTuner.vue` is the palette tuning panel, deliberately kept in the tree even while nothing mounts it — **do not delete it as dead code**. The user periodically remounts `<ColorTuner />` in `pages/[[side]].vue` and retargets its `TUNE_GROUPS` at another side's palette variables to pick that side's colors live: the panel renders per-group H/S/L delta sliders, overrides the target variables inline on `<html>` (so every var()-derived color follows), stops wheel/keydown so interacting never spins the cube, and shows the deltas plus resulting hexes to be read off a screenshot and baked back into `global.css`. Its `baseHex` values must mirror the current `:root` palette, or mounting it would repaint the page with stale colors. `components/GlitchTuner.vue` is its numeric sibling for the bottom side's glitch pass: SVG knobs write straight onto the `#interference-warp-filter` primitives (they can't consume CSS vars; the filter only exists while the bottom side is active, so «применить» re-applies after a side round-trip), CSS knobs override the band's and the scanlines' custom properties inline on `<html>` (`--band-*`, `--scanline-*`). Both tuners are currently unmounted (nothing in the tree renders them) with their defaults mirroring the baked values, ready to be remounted in `pages/[[side]].vue` for the next tuning session — like ColorTuner, GlitchTuner must not be deleted as dead code.
- `assets/css/global.css` is imported via `nuxt.config.ts` `css:` — `body` has `overflow: hidden` and a 500×500 minimum because the cube is a fixed-size viewport widget. Its `:root` block is the single source of truth for shared colors (`--color-*`) and the font pairs (`--font-*`): every shared color is defined exactly once there; consumers reference the var or derive alpha/tint variants via CSS relative color syntax (`rgb(from …)`) and `color-mix()` — plain `#fff` is the only color kept inline. The `SIDE_COLORS`/`SIDE_BACKGROUNDS` maps in `utils/sides.ts` hold `var(--color-…)` references, not hex.

There is **no FSD structure in use** (commit `9a11a3e removed FSD` explicitly reverted it). Do not reintroduce `app/widgets/features/entities/shared/` directories unless the user asks — put new work under `components/`, `pages/`, etc.

## Code style specifics (beyond the user-global rules in `~/.claude/CLAUDE.md`)

The ESLint + Stylelint configs here are strict and opinionated. Run `npm run lint` after edits; do not manually reformat against these rules if lint will auto-fix them.

- **TypeScript**: `tsconfig.json` enables `noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`, `noPropertyAccessFromIndexSignature`, and `noImplicitOverride`. Array/record accesses return `T | undefined` — narrow before use (see `TheCube.vue` guards on `nextSide`).
- **ESLint**: `@typescript-eslint/explicit-function-return-type` is on — always annotate return types on functions. `no-explicit-any` is an error (use `unknown`). `consistent-type-definitions` requires `interface` over `type` for object shapes. `padding-line-between-statements` enforces blank lines around `const`/`let`/`return`/control-flow blocks — most whitespace "weirdness" you see is mandated.
- **Vue template rules**: `component-name-in-template-casing: PascalCase`, `attribute-hyphenation: always`, strict attribute ordering (`DEFINITION → LIST_RENDERING → CONDITIONALS → … → EVENTS`), and `first-attribute-linebreak: below` for multiline elements.
- **Stylelint `order/properties-order`**: CSS properties must follow a fixed order (positioning → box model → typography → background → border → effects → animation → cursor → content). See `.stylelintrc.json` for the full list. Also: `color-named: never` (use hex), `declaration-no-important: true`, strings use double quotes.
